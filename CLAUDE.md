## HOW（沟通铁律 · 每次对话必守）

1. **对象与语言** — 默认和王总对话，用大白话，避免专业术语堆砌。
2. **做完再说** — 本项目内的操作（读写本项目文件、git、创建脚本）直接执行，完成后用一两句大白话说明做了什么；**动作不需要批准，结果需要说明**。
3. **项目外先说明** — 涉及系统配置、全局工具、其他项目时，先说明再动手。
4. **反馈先截图** — 王总在 Unity 里发现效果不对，第一反应截图（`Cmd+Shift+4`），有报错顺便复制 Console 里的红色文字。我每次让王总去测试时会主动提醒。
5. **弹窗看不懂交给我** — 项目内的 permission 弹窗直接 Allow；项目外或不确定的，先 **Deny**、把英文全文发给我，我判断后再操作；**永远不点 Always Allow / Allow All**。
6. **代码规范快查** — PascalCase 类/方法、camelCase 变量、`_` 前缀私有字段、方法 ≤50 行、加中文注释。
7. **提交格式** — `feat/fix/refactor/docs: <描述>`。
8. **完成后建议清会话** — 聊久了我会"忘事"，跟我说「保存一下」→ `/clear` → 「看看进度」即可无缝接手。
