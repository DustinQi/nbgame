# 技能选择指南

> 本文件是 CLAUDE.md 的外链细则，列出**各类开发场景对应的 Skill 选择**。
> 触发条件：需要选择合适的技能/工具来完成任务时。
>
> 注：技能名保持英文形式（原始命名），王总可用中文触发调用（见 `自然语言命令映射.md`）。

---

## 规划与设计

| 场景 | 可选 Skills | 说明 |
|------|------------|------|
| 功能规划 | `superpowers:writing-plans` / `ecc:planner` | 制定实施计划 |
| 系统架构 | `ecc:architect` | 架构决策和设计 |
| 需求梳理 | `superpowers:brainstorming` | 和王总一起探索需求 |

## 开发与实现

| 场景 | 可选 Skills | 说明 |
|------|------------|------|
| 分步开发 | `superpowers:subagent-driven-development` | 复杂功能拆解实现 |
| 并行任务 | `superpowers:dispatching-parallel-agents` | 多任务同时推进 |
| 重构清理 | `ecc:refactor-clean` | 代码清理和重复消除 |

## 质量保障

| 场景 | 可选 Skills | 说明 |
|------|------------|------|
| 代码审查 | `ecc:code-review` / `superpowers:requesting-code-review` | 提交前审查 |
| 安全审查 | `ecc:security-review` | 安全检查 |
| 隐蔽 Bug | `ecc:silent-failure-hunter` | 排查沉默错误 |
| 性能优化 | `ecc:performance-optimizer` | 性能瓶颈分析 |
| 构建修复 | `ecc:build-error-resolver` | 构建失败排查 |
| 最终验证 | `superpowers:verification-before-completion` / `verification-loop` | 完成前确认 |

## 流程与协作

| 场景 | 可选 Skills | 说明 |
|------|------------|------|
| TDD 开发 | `ecc:tdd-guide` | 测试驱动开发 |
| 分支收尾 | `superpowers:finishing-a-development-branch` | 完成任务并提交 |
| 发求助邮件 | `email-ops` | 向 Dustin 发送邮件 |

## 自动遵循的规则体系

以下 ECC 规则会在我开发时自动生效，无需手动调用：

- **`ecc:common/coding-style.md`** — 编码风格规范（不可变性、KISS、DRY、YAGNI）
- **`ecc:common/git-workflow.md`** — Git 提交和 PR 工作流
- **`ecc:common/testing.md`** — 测试要求（80%+ 覆盖率）
- **`ecc:common/security.md`** — 安全检查清单
- **`ecc:common/code-review.md`** — 代码审查标准
- **`ecc:common/development-workflow.md`** — 完整开发流程
