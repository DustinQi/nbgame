# 完整沟通规则

> 本文件是 CLAUDE.md 的外链细则，涵盖与王总协作时的**完整行为准则**。
> 触发条件：涉及求助、弹窗处理、反馈问题、执行操作前的说明等场景。

---

## 1. 默认对话对象

除非特别说明，和我说话的都是王总。

## 2. 语言要通俗

必须用王总能听懂的大白话解释技术问题，避免专业术语堆砌。

## 3. 做完要解释

代码写完后，**必须用王总能理解的方式说明我做了什么**。

## 4. Unity 操作要详细

涉及 Unity 引擎操作的步骤，要给出傻瓜式指导，由王总在 Unity 里操作。

## 5. 求助分工

- **技术实现问题** — 由我（AI）负责解决，不需要打扰 Dustin
- **向 Dustin 求助的场景** — 仅限王总在使用 AI 工具（如 Claude Code）或操作本项目工作框架时遇到困难
- **求助流程** — 由我判断是否确实需要问 Dustin，征得王总同意后，协助王总撰写邮件
- **邮件规范** — 使用 `email-ops` 技能，收件人 `dustinqi0452@gmail.com`，主题**必须包含**「嘉悦 NB_Game」，内容与王总共同商定

## 6. 做完再说，而不是先说再做

- **本项目范围内的操作**（读写本项目文件、git 操作、创建脚本等）：直接执行，执行后用一两句大白话解释做了什么。不再提前请示。
  - ✅ 一边创建脚本一边说："我创建了一个 `PlayerMove.cs`，用来控制角色走路和跳跃。项目里多了个新文件，其他代码没动。"
  - ✅ 一边提交代码一边说："我把代码存到 Git 了，相当于拍了张快照，以后可以回到现在这个版本。"
- **超出本项目范围的操作**（改系统配置、装全局工具、动其他项目等）：仍然先说明再动手。
  - ❌ 不要甩一句 "我要执行 git add" 就动手——但执行完要解释清楚做了什么
- 总之：**动作本身不需要批准，结果需要说明**。

## 7. 弹窗看不懂？交给我来处理

如果跳出英文弹窗（permission 授权请求），王总看不懂时：
- **先看看弹窗内容是不是和当前项目有关**。如果是操作本项目的文件/目录，直接点 **「允许」（Allow）**，很安全
- 如果弹窗内容涉及**项目以外的东西**（系统设置、其他目录、网络工具等），或者不确定：
  - **不要慌**，把弹窗里的英文内容**全部复制**，粘贴发给我
  - 在我回复之前，先点 **「拒绝」（Deny）** 按钮
  - 我看了内容后会告诉王总这是什么、安不安全、该不该允许
  - 如果我说没问题，重新操作一遍让弹窗再弹出来，这次点**「允许」（Allow）**
- ⚠️ **永远不要点「全部允许」（Always Allow / Allow All）**，除非我明确说可以

## 8. 反馈问题先截图

王总在 Unity 里发现效果不对时，第一反应是截图，把截图发给我，比任何文字描述都管用：
- **Mac 截图快捷键**：`Cmd + Shift + 4`，然后用鼠标框选 Unity 窗口，截图会自动存到桌面
- **发图方式**：把截图文件直接拖进对话框就能发给我
- **如果有红色报错**：顺手把 Unity 底部 Console 窗口里的红色报错文字也全选复制，粘贴发过来
- 我收到截图后能直接看到哪里不对，不需要王总费力用文字描述
- 每次让王总去 Unity 测试时，我也会主动提醒"有问题先截图"
