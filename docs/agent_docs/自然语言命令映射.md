# 自然语言命令映射

> 本文件是 CLAUDE.md 的外链细则，记录**王总日常口语与技能调用的对应关系**。
> 触发条件：识别王总的意图并选择对应技能时。
>
> 原则：**技能名保持英文，王总用中文触发**。
> 说明：下表映射不是死板的，只要王总说的话意图接近，就自动执行对应行为，不需要精确关键词。

---

## 映射表

| 王总说的话（举例） | 相当于执行 | 效果 |
|---|---|---|
| "dustin咱们聊聊"、"聊聊天"、"有个想法" | `/聊聊` | 使用 `superpowers:brainstorming` 一起讨论创意 |
| "开工了"、"开始干活"、"开始做这个" | `/开工` | 开始编码实现 |
| "做到哪了"、"看看进度"、"现在啥情况" | `/进度` | 读取 `docs/项目进度.md` 汇报状态 |
| "帮我检查下代码"、"看看对不对"、"查查bug" | `/检查` | 使用 `ecc:code-review` 审查代码 |
| "出问题了"、"搞不定了"、"报错了"、"出错啦" | `/救命` | 使用 `superpowers:systematic-debugging` 排查问题 |
| "保存一下"、"提交代码"、"存个档" | `/保存` | Git 提交 + 更新进度文档。如果感觉我反应变慢可以顺便开个新会话 |
| "帮我找dustin"、"联系dustin"、"帮我问问dustin" | `/找Dustin` | 判断后协助邮件求助 |
| "帮我规划"、"计划一下"、"这个怎么做" | `/规划` | 使用 `superpowers:writing-plans` 制定计划 |
| "代码太乱了"、"整理一下"、"优化代码" | `/整理` | 使用 `ecc:refactor-clean` 重构 |
| "搞完了"、"做完了"、"收工"、"这个功能好了" | `/收工` | 使用 `superpowers:finishing-a-development-branch` 收尾。完成后建议开新会话保持流畅 |
| "试试好不好用"、"测试一下"、"试试看" | `/测试` | 生成测试步骤 |
| "记录一下"、"更新进度"、"记一笔" | `/记录` | 更新 `docs/项目进度.md` |
| "我该干啥"、"现在做什么"、"接下来干啥"、"不知道干啥了" | `/干啥` | 读取进度文档并给出下一步建议 |
